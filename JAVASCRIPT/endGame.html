The server found the requested resource and sent it back to the client. This is the most common successful response for GET requests. Example: Successfully loading a webpage or retrieving data from an API. 2. 404 NOT FOUND (Client Error): The server cannot find the requested resource. The URL is incorrect or the page/file doesn't exist on the server. Example: Typing a wrong URL or clicking a broken link. 3. 500 INTERNAL SERVER ERROR (Server Error): The server encountered an unexpected condition that prevented it from fulfilling the request. Often caused by server-side programming errors. Example: PHP syntax error, database connection failure, or server misconfiguration. 4. 301 MOVED PERMANENTLY (Redirection): The requested resource has been permanently moved to a new URL. The browser should update its bookmarks and use the new URL in future requests. Example: Website domain change or URL restructuring. 5. 403 FORBIDDEN (Client Error): The server understands the request but refuses to authorize it. The client doesn't have permission to access the resource. Example: Trying to access admin pages without proper authentication.
